# Known Code-Assist Patterns

Append-only list. Each entry records a recurring review category, its pattern,
and the canonical fix or dismissal rationale.

- MODEL ROUTING · Do not substitute a model merely because it shares a backend. · Select within the requested role and required capability set.
- FAILURE TAXONOMY · Do not label schema, safety, or quality rejection as a fleet outage. · Classify and quarantine per job and model.
- PRUNING · Never delete incumbent, fallback, rollback, pinned, shared, or loaded models. · Recompute protection immediately before LRU deletion; upstream availability is not a deletion gate.
